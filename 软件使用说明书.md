## 1. 引言

### 1.1 编写目的
本说明书旨在详细介绍“AI 美食食谱推荐系统 V1.0”的功能规格、安装部署步骤及用户操作流程。本文档供软件测试人员、最终用户及系统维护人员阅读，以确保用户能够正确、高效地使用本软件解决美食识别与食谱查询需求。

### 1.2 项目背景
随着社交媒体和数字生活的普及，人们越来越习惯通过图片分享美食。然而，从一张美食图片到获取其具体的烹饪方法，往往需要繁琐的搜索过程。本软件基于深度学习与自然语言处理技术，构建了一个端到端的智能系统，连接“视觉感知”与“知识检索”，用户只需上传一张图片，系统即可自动识别食物类别，并推荐详细的烹饪食谱，实现“所见即所得”的烹饪辅助体验。

### 1.3 术语定义
*   **Food-101**：本系统使用的基础视觉数据集，包含 101 种常见美食类别。
*   **EfficientNet-B3**：本系统集成的卷积神经网络模型，用于提取图像特征并进行分类。
*   **Sentence-BERT**：用于计算文本语义相似度的预训练模型，负责食谱检索。
*   **置信度 (Confidence)**：模型对当前识别结果确信程度的量化指标，范围 0~100%。

### 1.4 参考资料
1.  《计算机软件产品开发文件编制指南 GB/T 8567-2006》
2.  Food-101 Dataset Paper (ECCV 2014)
3.  EfficientNet Paper (ICML 2019)

## 2. 软件概述

### 2.1 软件目标
本软件致力于为美食爱好者提供一个便捷、智能的食谱获取工具。通过上传图片，快速识别食物种类（支持 101 种中西美食），并匹配最正宗的烹饪食谱，同时通过内置翻译引擎消除语言障碍。

### 2.2 功能简介
本软件主要包含以下四大功能模块：
1.  **视觉识别模块**：利用深度学习模型对上传的美食图片进行分析，输出最可能的食物名称及其概率。
2.  **智能检索模块**：基于语义向量匹配技术，从百万级食谱库中检索出与识别结果最相关的 Top-5 食谱。
3.  **交互展示模块**：提供卡片式食谱展示界面，支持折叠/展开原料与步骤，界面美观流畅。
4.  **多语言服务模块**：集成百度翻译 API，提供“菜名翻译”和“食谱全文翻译”功能，支持中英互译。

### 2.3 性能指标
*   **识别准确率**：Top-1 准确率约为 80.05%，Top-5 准确率超过 95%。
*   **响应时间**：
    *   图片上传与预览：< 100ms
    *   模型推理与检索：< 500ms（基于 CPU 环境）
    *   翻译响应：约 1.2s/条（受 API 限流策略控制）
*   **并发能力**：支持多用户同时访问 Web 界面（受限于服务器硬件配置）。

### 2.4 运行环境

#### 2.4.1 硬件环境
*   **处理器 (CPU)**：Intel Core i5-8400 / AMD Ryzen 5 2600 及以上（推荐支持 AVX2 指令集）。
*   **内存 (RAM)**：建议 8GB 及以上（模型加载需占用约 1GB 内存）。
*   **硬盘**：至少 2GB 可用空间（用于存放模型权重文件和食谱数据库）。
*   **显卡 (GPU)**：可选。如有 NVIDIA 显卡并安装 CUDA 环境，推理速度可提升 5-10 倍。

#### 2.4.2 软件环境
*   **操作系统**：Windows 10/11 (64位), macOS 10.14+, Linux (Ubuntu 20.04+)。
*   **运行依赖**：Python 3.8 或更高版本。
*   **浏览器**：Google Chrome 80+, Microsoft Edge 88+, Firefox 78+。

## 3. 安装与初始化

### 3.1 软件安装
本软件为绿色免安装版本，用户只需解压即可使用。

1.  **获取软件包**：将软件压缩包下载至本地磁盘。
2.  **解压文件**：解压到非中文路径下（推荐），例如 `D:\AI_Food_Recommender`。
3.  **安装依赖库**：
    *   打开命令提示符（CMD）或 PowerShell。
    *   进入软件根目录。
    *   执行以下命令安装 Python 依赖库：
        ```bash
        pip install -r requirements.txt
        ```
    *   *注：请确保网络连接正常，以便下载 PyTorch 等第三方库。本系统依赖 `python-multipart` 处理图片上传，依赖 `fastapi` 和 `uvicorn` 运行 Web 服务。*

### 3.2 系统初始化与启动
1.  **启动服务**：
    在软件根目录下，双击运行 `run.bat`（如有）或在终端输入：
    ```bash
    python app.py
    ```
2.  **服务加载**：
    终端将显示如下日志，表示正在加载 AI 模型和数据库，请耐心等待 10-30 秒：
    ```text
    Initializing application...
    ✅ Loaded 101 food classes.
    ✅ Loaded recipes for 101 categories.
    ✅ Model loaded successfully on cpu
    Uvicorn running on http://0.0.0.0:8000
    ```
3.  **访问系统**：
    打开浏览器，在地址栏输入 `http://localhost:8000`，按回车键即可进入系统主界面。

### 4.5 跨语言智能翻译
针对英文基础较弱的用户，系统提供两级翻译功能。

#### 4.5.1 菜名翻译
**操作步骤**：
1.  在“AI 识别结果”区域，点击英文菜名旁的黄色 **“翻译”** 按钮。

**预期结果**：
*   英文单词（如 `Peking Duck`）瞬间变为中文（`北京烤鸭`）。

#### 4.5.2 食谱全文翻译
**操作步骤**：
1.  在食谱卡片右上角，点击灰色的 **“翻译食谱”** 按钮。
2.  观察按钮状态变为“翻译中...”。

**预期结果**：
1.  系统依次翻译标题、原料和步骤（中间可能有 1-2 秒间隔，为系统自动限流保护）。
2.  翻译完成后，卡片内的所有英文文本将被替换为中文。
3.  系统会自动展开该卡片，方便用户直接阅读中文制作方法。

## 5. 异常处理与故障排除

### 5.1 常见报错信息

| 错误现象 | 可能原因 | 解决方法 |
| :--- | :--- | :--- |
| **500 Internal Server Error** | 后端服务未启动或报错 | 检查终端窗口是否有报错日志，重启 `app.py`。 |
| **翻译失败 / Translation Error** | 网络不通或 API 配额耗尽 | 检查本机网络；等待几分钟后重试（触发了 QPS 限制）。 |
| **点击上传无反应** | 浏览器兼容性问题 | 建议使用最新版 Chrome 或 Edge 浏览器。 |
| **识别结果为 "frozen_yogurt"** | 模型对纯色/无特征图片误判 | 请上传主体清晰、光线明亮的美食特写图片。 |

### 5.2 故障排除指南
1.  **无法启动服务**：
    *   检查是否已安装 Python 3.8+。
    *   检查 `requirements.txt` 中的依赖是否全部安装成功。
    *   检查端口 `8000` 是否被其他程序占用。
2.  **页面一直显示“Loading”**：
    *   按 `F12` 打开浏览器控制台，查看 `Console` 面板是否有红色报错。
    *   尝试清除浏览器缓存或使用无痕模式重新访问。
